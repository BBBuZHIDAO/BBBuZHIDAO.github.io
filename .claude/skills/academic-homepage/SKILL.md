# Academic Homepage Skill

Build a static academic personal website for GitHub Pages.

## Core positioning

The site should present the person as both:
- an academic researcher in nanoscale heat transfer, interfacial thermal transport, two-dimensional materials, van der Waals materials, phonon transport, molecular dynamics, and machine-learning potentials;
- a scientific software developer contributing to GPU-accelerated molecular dynamics and high-performance atomistic simulation software.

## Content rules

Use `content/profile.md` as the single source of truth.

Do not invent:
- publications
- DOI links
- GitHub links
- Google Scholar links
- journal status
- awards
- metrics
- project dates

Do not include:
- VDWToolkit
- Guanghe Funding / domestic accelerator card testing project
- phone number
- private notes
- unverified information from older CV versions

## Site sections

Recommended sections:
1. Home
2. Research
3. Development
4. Publications
5. Experience
6. CV
7. Contact

## Writing style

Use concise academic English.

Preferred tone:
- precise
- understated
- technical
- confident but not promotional

Avoid:
- exaggerated adjectives
- marketing language
- vague claims
- AI-template language
- overlong paragraphs

## Design style

The design should be:
- academic
- minimal
- polished
- clear
- responsive
- easy to maintain

Avoid:
- flashy animations
- particle backgrounds
- excessive gradients
- too many icons
- cluttered cards
- low-contrast text

## Technical rules

Use:
- plain HTML
- CSS
- minimal JavaScript

Keep:
- relative paths
- semantic HTML
- readable section IDs
- GitHub Pages compatibility
- mobile-first responsive layout
